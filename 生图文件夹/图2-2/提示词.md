## 首选：以 `aios_fig2.png` 为底图修改

在这张图的基础上修改，严格保持原图的视觉风格：保留虚线分隔的水平分层布局、每层右侧的斜体层名标注、层内彩色圆角小模块、横贯整层的长条"总线"模块、左侧独立的灰底支撑栏，以及带文字标签的黑色细双向箭头。只按以下要求修改内容与结构：

### 整体布局（自上而下五个虚线分区 + 左侧贯通支撑栏）

**最顶层「应用服务与执行层（执行层）」**（对应原图 Agent Application 绿色横条）：
- 绿色长条容器，标题「应用服务」，内部并排五个白底小胶囊（每个带小图标）：「政策推演应用」「交通治理应用」「环境评估应用」「应急演练应用」「科研实验应用」；
- 容器下方居中放一个蓝色长条模块「数据服务 API / 用户界面 UI-UX」（对应原图 AIOS-Agent SDK 的位置）；
- 从五个应用胶囊到这个蓝色模块画细箭头汇聚；蓝色模块左右两侧各引出一条带标签的箭头：左侧标签「业务流程请求」，右侧标签「推演结果返回」。

**第二层「智能决策与仿真层（决策层）」**（对应原图 AIOS Kernel 大蓝底容器）：
- 一个浅蓝色大容器，标题「EcoCitySociety 仿真决策核心」；
- 容器顶部横贯一条紫色长条总线，写「统一调度与消息总线」；
- 总线下方一排七个彩色圆角小模块（配色沿用原图那排 Manager 模块）：「多智能体系统」「环境仿真器库」「MARL 训练器」「优化调度器」「因果评估器」「决策支持器」「结果溯源器」；
- 每个小模块与紫色总线之间画竖直双向小箭头（保留原图画法）；
- 容器底部再横贯一条白底长条「仿真运行时（Runner：多进程 / Ray / Dask）」，与上排模块用一条竖直双向箭头连接。

**第三层「数据处理与分析层（处理层）」**：
- 白底容器内从左到右四个圆角模块形成流水线：「数据清洗」→「异常值检测与去除」→「ML/DL 数据建模」→「预测分析与风险评估」，模块间用带箭头的实线依次连接；
- 流水线整体与上层"仿真运行时"之间画一条竖直双向箭头，标签「特征与指标供给」。

**第四层「数据采集层（感知层）」**（对应原图 Hardware 灰色横条）：
- 灰底长条容器，标题「多源感知接入」，内部并排五个白底小模块：「IoT 传感设备」「移动终端」「外部 API」「气候/交通/社会经济数据库」「流式数据接入」；
- 每个小模块向上引一条细箭头汇入第三层的「数据清洗」模块，汇聚处标签「原始数据流」。

**左侧贯通支撑栏「监控与反馈层（监控层）」**（对应原图左侧灰底 OS Kernel 竖栏，纵向跨越第二、三层）：
- 灰底竖栏，标题「监控与反馈」，自上而下四个白底小模块：「系统状态监控」「数据质量监控」「异常预警」「自我优化」；
- 四个小模块用右侧的竖向汇流线连到主体各层（保留原图 OS Kernel 内部的汇流箭头画法）：「系统状态监控」连到第二层总线、「数据质量监控」连到第三层流水线、「异常预警」连到最顶层应用条、「自我优化」连到第二层运行时；
- 竖栏底部向下引一条箭头连到第四层感知条，标签「采集调度」。
